# О проекте

Контроль выкладки товаров является одной из важных задач в сфере розничной торговли. Правильное расположение продукции на полках влияет на удобство покупателей, соблюдение корпоративных стандартов и эффективность продаж. На практике проверка выкладки чаще всего выполняется вручную, что требует значительных временных затрат и не исключает вероятность ошибок.

Проект **Shelf Analyzer** разработан для автоматизации данного процесса. Приложение позволяет сравнить фотографию торговой полки с заранее подготовленной планограммой и определить, насколько фактическое расположение товаров соответствует ожидаемому.

В качестве инструмента анализа изображений используется VISION API, способная распознавать содержимое фотографии и формировать структурированное описание расположения товаров. Полученный результат автоматически сопоставляется с планограммой, после чего пользователь получает подробный отчет о выявленных несоответствиях.

Приложение реализовано в виде веб-сервиса на платформе Streamlit и не требует установки дополнительного программного обеспечения со стороны пользователя. Для работы достаточно открыть приложение в браузере, создать или загрузить планограмму и выполнить анализ фотографии.

---

# Основные возможности

В приложении реализованы следующие функции:

* создание новых планограмм с произвольным количеством полок;
* редактирование существующих планограмм;
* сохранение и загрузка планограмм в формате JSON;
* загрузка фотографий торговых полок;
* анализ изображений с использованием мультимодальной модели;
* сравнение результатов анализа с эталонной планограммой;
* определение отсутствующих товаров;
* выявление товаров, отсутствующих в планограмме;
* проверка правильности порядка расположения продукции;
* вычисление общего процента соответствия выкладки;
* отображение детального отчета по каждой полке;
* сохранение истории выполненных анализов.

# Принцип работы

Работа приложения состоит из нескольких последовательных этапов.

На первом этапе пользователь формирует планограмму, которая представляет собой описание правильной выкладки товаров. Для каждой полки задается список товаров в том порядке, в котором они должны располагаться.

После подготовки планограммы пользователь загружает фотографию торговой полки. Вместе с изображением планограмма передается модели API, которая анализирует содержимое фотографии и определяет расположение товаров.

Полученный ответ преобразуется в структурированный формат и передается модулю сравнения. Приложение последовательно сопоставляет фактическую выкладку с эталонной, определяя наличие отсутствующих товаров, лишних позиций, а также нарушения порядка размещения.

По завершении анализа формируется итоговый отчет, содержащий процент соответствия выкладки, перечень найденных товаров и список выявленных несоответствий.

# Структура приложения



---

## Работа с планограммами

Перед выполнением анализа пользователь создает планограмму — описание эталонной выкладки товаров.

Для каждой полки можно задать список товаров в требуемом порядке, после чего сохранить результат в формате JSON. При необходимости ранее созданные планограммы могут быть повторно загружены и использованы без дополнительного редактирования.

---

## Анализ изображений

Раздел анализа является основным функциональным модулем приложения.

После загрузки фотографии выполняется обращение к API. Полученный результат автоматически преобразуется в удобный для дальнейшей обработки формат и сравнивается с выбранной планограммой.

После завершения анализа пользователю отображается подробный отчет с результатами проверки.

---

## История анализов

Каждый выполненный анализ сохраняется в истории приложения.

Это позволяет повторно просматривать результаты предыдущих проверок, сравнивать их между собой и отслеживать изменения состояния выкладки без необходимости повторной обработки изображений.

# Установка и запуск

Клонируйте репозиторий:

```bash
git clone https://github.com/EgorBubnov/shelf-analyzer.git
```

Перейдите в директорию проекта:

```bash
cd shelf-analyzer
```

Установите необходимые зависимости:

```bash
pip install -r requirements.txt
```

Запустите приложение:

```bash
streamlit run app.py
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8501
```
